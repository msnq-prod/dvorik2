# Инструкция по настройке и запуску проекта

Для корректной работы приложения необходимо выполнить предварительную настройку.

## 1. Настройка переменных окружения

Перед первым запуском скопируйте файл `.env.example` в новый файл `.env`:

```bash
cp v1/.env.example v1/.env
```

Затем откройте файл `v1/.env` и замените значения-заполнители (`your_...`) на ваши реальные данные.

### Описание переменных

-   `MYSQL_ROOT_PASSWORD`: Пароль для суперпользователя `root` в MySQL.
-   `MYSQL_DATABASE`: Название базы данных, которая будет создана.
-   `MYSQL_USER`: Имя пользователя для подключения к базе данных.
-   `MYSQL_PASSWORD`: Пароль для этого пользователя.
-   `DATABASE_URL`: Строка подключения к базе данных. **Обычно ее не нужно менять**, так как она использует другие переменные.
-   `TELEGRAM_TOKEN`: Токен вашего Telegram-бота. Получите его у [@BotFather](https://t.me/BotFather).

**Внимание:** Без правильного `TELEGRAM_TOKEN` сервис бота не сможет запуститься. Без правильных данных для MySQL не запустится бэкенд.

## 2. Запуск приложения

После того как вы настроили `.env` файл, вы можете запустить все сервисы с помощью Docker Compose. Выполните команду из корневой директории проекта:

```bash
docker compose -f v1/docker-compose.yml up --build
```

-   **Админ-панель** будет доступна по адресу: `http://localhost:8080`
-   **API бэкенда** будет доступно по адресу: `http://localhost:8000`
-   **Телеграм-бот** начнет принимать сообщения.
